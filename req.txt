2) Data Integration Blueprint (What to fetch & how)
Data domains & recommended sources
Accommodation (Hotels / Listings)

Sources: Booking.com Affiliate API, Agoda Affiliate, Expedia Rapid API, Airbnb (partner), HotelsCombined

Key data: hotel_id, name, address, lat/lng, images, room types, price, currency, availability, cancellation_policy, rating, amenities

Integration method: Affiliate API (REST) or OTA partner API. Cache inventory snapshots. Use webhooks for booking confirmations when supported.

Activities & Tours

Sources: Klook Affiliate, GetYourGuide API, Viator, Local tour operators via custom partner API

Key data: activity_id, title, location, duration, schedule, price, booking_url, images, min_people, reviews

Integration: OAuth/API keys; periodic sync + realtime on-demand pricing checks

Places / POI / Restaurants

Sources: Google Places API (places/nearby), OpenTripMap, Yelp, Local providers (Wongnai, LINE MAN)

Key data: place_id, name, kinds/categories, opening_hours, phone, images, price_level, rating, popularity_score

Integration: On-demand queries + cache popular city POIs daily

Transport (Flights / Trains / Buses / Car Rentals)

Sources: Skyscanner API, Amadeus, Kiwi, local bus APIs, car rental aggregators

Key data: routes, schedules, fares, seat_availability, booking_token

Integration: Use GDS / aggregator for flights; rapid lookup + redirect to partner for booking

Weather

Sources: OpenWeatherMap, WeatherAPI

Use: to tailor itinerary (indoor/outdoor), notify on rainy days

Integration: on-demand forecast calls when generating itinerary

Currency / Pricing / Fees

Sources: exchangerate.host, CurrencyLayer

Use: convert prices, show total budget in user's currency

Reviews & UGC

Sources: TripAdvisor (if partner), Google Reviews, in-app user reviews

Use: ranking, trust signals

User Data

Data: preferences, past trips, saved places, ratings

Storage: PostgreSQL (encrypted columns), Analytics (Mixpanel / GA4)

Integration patterns
Aggregator Layer: implement a thin service per provider ("Provider Adapter") that normalizes provider responses to an internal schema (Hotel, Activity, POI).

Caching: Redis for hot POIs; daily snapshot for hotels/room inventory in Postgres for commonly searched destinations.

On-demand enrichment: when user requests itinerary, fetch live prices from affiliate APIs for concrete booking links.

Rate-limits handling: exponential backoff + queueing (RabbitMQ / Kafka) for heavy traffic.

Webhook handling: listen for booking confirmations from partners; reconcile state in DB.

3) AI Data Flow (from sources → AI → user)
Goals
สร้าง itinerary ที่ "ทำได้จริง" (feasible): เวลาเดินทาง, opening hours, distance, budget-aware

Personalized: ปรับให้เข้ากับสไตล์ผู้ใช้ (foodie, slow travel, photogenic)

Actionable: ให้ลิงก์จองจริงได้ (affiliate links)

High-level steps
Input Collection (User + Context)

User input: destination(s), start_date, end_date, budget, travel_style, group_size, mobility constraints

User profile: past trips, saved places, preferences

Real-world data: POIs, hotels, transport options, weather forecast, opening hours

Preprocessing & Enrichment

Geospatial enrichment: compute distances and travel times between POIs (using Google Distance Matrix or routing engine)

Time constraints: filter POIs by opening_hours relative to user dates

Price normalization: convert all prices to user currency and apply estimated taxes/fees

Popularity & rating: compute a composite score per POI (rating, reviews, social signals)

Candidate Generation

Rule-based filters: must-have vs nice-to-have

Heuristic clustering: group POIs by proximity and time of day

Candidate pool: select N best candidates per timeslot (e.g., morning/afternoon/evening)

LLM Prompting / Planner

Create structured prompt containing: user constraints, candidate POIs (with lat/lng, opening_hours, duration, score), hotel location, travel_time matrix

Ask LLM to produce: day-by-day plan with time slots, travel times, estimated costs, and booking suggestions

Use temperature control & few-shot examples for consistent structure

Example prompt skeleton:

You are an expert travel planner. Input: {user_profile, dates, hotel_location, candidates, travel_time_matrix, budget}.
Output: JSON with days[], each day contains timeblocks {start,end,activity,place_id,estimated_cost,notes,booking_link}.
Constraints: do not schedule activities outside opening_hours; minimize travel time; keep daily cost <= budget/day.
Post-processing & Validation

Validate LLM output: ensure time windows respect opening_hours; check travel times; total cost within budget

If invalid, run correction step (mini-optimizer) or re-prompt LLM with errors

Ranking & Personalization

Re-rank activities based on user's preference weightings (e.g., food lover +2 weight on restaurants)

Add alternative suggestions (fallbacks if sold out or closed)

Actionable Output

Formatted itinerary (UI-ready) + estimated trip cost + booking links for hotels/activities

Export PDF / Shareable link

Feedback Loop

After trip, collect user feedback (ratings per activity) → feed into ML ranking model

Use interactions to fine-tune prompts and weighting

4) Technical Considerations & Recommendations
Hybrid AI approach: combine deterministic logic (opening hours, travel time) with LLM creativity. Do not rely solely on LLM for constraints.

Caching & Freshness: cache POI metadata for 24h; fetch live prices on-demand when user indicates intent to book.

Monitoring: OpenTelemetry + Loki + Jaeger for trace of itinerary generation; track prompt latency/cost.

Security: store API keys in secret manager; encrypt PII at rest; GDPR/PDPA compliance for user data

Scalability: serverless functions for prompt calls + queue for heavy batch jobs; autoscale on traffic spikes

CREATE TABLE user_reviews (
  id uuid PRIMARY KEY,
  user_id uuid REFERENCES users(id),
  poi_id uuid REFERENCES poi(id),        -- หรือ hotel_id / activity_id
  rating smallint CHECK (rating BETWEEN 1 AND 5),
  review_text text,
  images text[],                         -- เก็บ URL จาก S3
  created_at timestamptz DEFAULT now(),
  status varchar DEFAULT 'pending'       -- pending / approved / rejected
);

CREATE TABLE poi_ratings_summary (
  poi_id uuid PRIMARY KEY,

### 🧩 Core Feature Set (TripAdvisor-style)

#### **1. Destination & Attraction Explorer**
- List top destinations by country, region, or theme (e.g. beaches, food trips, nature)
- Each destination page includes:
  - Hero image, map, and short AI-written summary
  - Top-rated attractions, hotels, and restaurants nearby
  - Filters: “Most Popular”, “Hidden Gems”, “Family-friendly”
  - “Things to Do” section with categories (tours, landmarks, local experiences)

#### **2. Hotel & Accommodation Listings**
- Display hotels with:
  - Name, image, star rating, price range, amenities
  - Link to booking (Agoda/Booking.com/Expedia API)
- Add filters: price, rating, location, type (resort, hostel, villa)
- Include review highlights:
  - “AI Summary: Guests loved the location and breakfast.”
- Sort options: Best Value / Top Reviewed / Distance

#### **3. Restaurant & Food Guide**
- Curate restaurants near attractions or hotels
- Include tags: Local Food, Fine Dining, Street Food
- User reviews + average ratings
- AI highlights for “Best Dishes” and “Traveler Favorites”

#### **4. Reviews & Ratings System**
- Allow verified users to post text + photo reviews
- 1–5 star rating system for: cleanliness, service, value, food quality, etc.
- Add AI Review Summarizer:
  - Analyze review tone (positive, neutral, negative)
  - Display summary badges (“Most mentioned: Clean rooms, Great location”)
- Add “Was this review helpful?” voting feature

#### **5. Community Feed (Social Experience)**
- Show recent posts from travelers (“Just visited Chiang Mai!”)
- Allow users to upload trip photos & stories
- Add like/comment/share system
- Highlight top contributors with badges and ranks

#### **6. Trip Planner & Saved Lists**
- Users can save hotels, restaurants, or attractions to “My Trips”
- AI-assisted itinerary builder:
  - Suggests optimal routes and time allocation
  - Syncs with saved items automatically
- Option to export trip plan to PDF or share link

#### **7. Leaderboards & Gamification**
- Reward active users for:
  - Writing reviews, uploading photos, getting upvotes
- Display top reviewers by week/month
- Unlock badges: “Top Explorer”, “Food Lover”, “Hotel Guru”

#### **8. Map Integration**
- Interactive map for browsing attractions and hotels
- Cluster pins for high-density areas
- Hover tooltip → show rating + photo preview
- Option to “Plan on Map” for itinerary view

#### **9. AI Recommendation Engine**
- Personalized feed (“Recommended for You”)
- Suggest destinations or hotels based on:
  - User history, preferences, and review patterns
- Generate AI trip suggestions (“Best 3-day trip for couples in Phuket”)
- Combine review sentiment + location data for smarter ranking

#### **10. Multi-language & Localization**
- Support Thai, English, Chinese, Japanese
- Localize place names and currency display
- Auto-detect region from IP or profile

---

### 📊 Optional Analytics & Admin Features
- Dashboard for tracking most reviewed places
- Review moderation tools
- AI-assisted spam/fake review detection
- Sentiment trend analytics (positive % by location)

---
